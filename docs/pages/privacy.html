Privacy

The web edition never pushes to NetBox, and never sees your token.

No config you paste is stored. No NetBox URL or token is ever sent to this service. The model and the import script are generated and handed back to you — that is the whole interaction.

Why no live push

NetBox lives on a private network the Cloudflare edge cannot reach, and an API token is sensitive. Pushing from a public site would mean sending an admin token through a third party — exactly the thing a security-minded operator should refuse.

What you get instead

A downloadable import.py that contains your parsed model inline. You run it yourself, on your own host, against your own NetBox:

NETBOX_URL=http://netbox.local:8080 NETBOX_TOKEN=xxxx python3 import.py

The token is read from your environment and never leaves your machine.

Súkromie

Webová edícia nikdy nepushuje do NetBoxu a nikdy nevidí tvoj token.

Žiadny config, ktorý vložíš, sa neukladá. Žiadna NetBox URL ani token sa nikdy neodošlú tejto službe. Model a import skript sa vygenerujú a vrátia tebe — to je celá interakcia.

Prečo žiadny živý push

NetBox žije v privátnej sieti, na ktorú Cloudflare edge nedosiahne, a API token je citlivý. Push z verejnej stránky by znamenal posielať admin token cez tretiu stranu — presne to, čo by bezpečnostne uvažujúci operátor mal odmietnuť.

Čo dostaneš namiesto toho

Stiahnuteľný import.py, ktorý obsahuje tvoj naparsovaný model priamo v sebe. Spustíš ho sám, na svojom vlastnom hoste, proti svojmu vlastnému NetBoxu:

NETBOX_URL=http://netbox.local:8080 NETBOX_TOKEN=xxxx python3 import.py

Token sa načíta z tvojho prostredia a nikdy neopustí tvoj stroj.
