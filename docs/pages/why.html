Why a parser

A running device's config is the most authoritative description of how it is set up. Use it.

A network device knows everything about itself — interfaces, addresses, VLANs, VRFs, routing, authentication. But that knowledge lives only on the box, as flat text in a vendor dialect. Building an inventory, an IPAM or an audit by hand means retyping what the device already knows, and the result is stale the moment it is written.

CiscoParser reads the config the way an engineer does — understanding the hierarchical parent/child structure, not just lines — and returns a structured model you can keep, diff and import. The web edition makes that a single paste, with nothing to install and nothing stored.

Where it fits

This is an import / documentation tool, not a configuration manager. It pairs with a standalone NetBox deployment: NetBox is the store, CiscoParser is one of the doors data comes in through.

Načo parser

Config bežiaceho zariadenia je najautoritatívnejší popis toho, ako je nastavené. Využi ho.

Sieťové zariadenie o sebe vie všetko — rozhrania, adresy, VLAN-y, VRF-y, smerovanie, autentifikáciu. Lenže táto znalosť žije len na boxe, ako plochý text vo vendorskom dialekte. Budovať inventár, IPAM či audit ručne znamená prepisovať to, čo zariadenie už vie, a výsledok je neaktuálny v okamihu, keď vznikne.

CiscoParser číta config tak ako inžinier — rozumie hierarchickej štruktúre rodič/dieťa, nielen riadkom — a vráti štruktúrovaný model, ktorý vieš uchovať, porovnať aj importovať. Webová edícia z toho robí jediné vloženie, bez inštalácie a bez ukladania.

Kam zapadá

Toto je nástroj na import / dokumentáciu, nie správca konfigurácie. Páruje sa so samostatným NetBox nasadením: NetBox je úložisko, CiscoParser sú jedny z dverí, ktorými dáta vstupujú.
